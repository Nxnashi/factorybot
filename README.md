# Учёт производства — Telegram Mini App

Мини-апп для учёта производства по этапам (приход ТМЦ → замес → формовка → QC → печь → готовая продукция) с выгрузкой в Excel.

## Что внутри

- `server.js` — express-сервер
- `src/db.js` — SQLite-схема (файл `factory.db` создаётся автоматически)
- `src/bot.js` — телеграм-бот, кнопка запуска мини-аппа
- `routes/stages.js` — API для рабочих (получить свой этап, номенклатуру, отправить данные)
- `routes/admin.js` — API админки (сотрудники, номенклатура)
- `routes/export.js` — генерация Excel-отчёта
- `public/index.html` — форма для рабочего
- `public/admin.html` — панель администратора (сотрудники / номенклатура / экспорт)

## Как задеплоить (Render + GitHub, как в прошлый раз)

1. Залить эту папку в новый GitHub-репозиторий.
2. На Render: New → Web Service → подключить репозиторий.
   - Build command: `npm install`
   - Start command: `npm start`
3. В Environment добавить переменные (см. `.env.example`):
   - `BOT_TOKEN` — токен от @BotFather
   - `WEBAPP_URL` — URL, который даст Render после первого деплоя (например `https://factory-tracker.onrender.com`), проставить и передеплоить
   - `ADMIN_TELEGRAM_ID` — твой Telegram ID (узнать через `/myid` у бота или @userinfobot)
   - `ADMIN_FULL_NAME` — твоё имя
4. У @BotFather: `/setmenubutton` или просто отправить `/start` боту — там уже есть кнопка запуска мини-аппа.
5. Зайти в бот под аккаунтом из `ADMIN_TELEGRAM_ID` → `/admin` → добавить остальных сотрудников по их Telegram ID и назначить этапы.

## Важно про бесплатный тариф Render

На бесплатном плане сервис засыпает после 15 минут простоя и потом ~30-50 сек стартует по первому запросу — рабочие могут увидеть "Загрузка…" чуть дольше на первый заход в смену. Если это критично, можно поставить платный план ($7/мес) или дергать `/health` каждые 10 минут внешним крон-сервисом (например cron-job.org) — бесплатно.

**Важно:** SQLite-файл `factory.db` на Render хранится на **эфемерном диске** — при передеплое/пересоздании контейнера данные удалятся. Для продакшена нужно подключить Render Disk (постоянное хранилище, есть в бесплатном плане с ограничением объёма) и указать в нём путь к файлу базы — либо сразу перейти на Postgres, если данных будет много. Для MVP и обкатки — сойдёт как есть, но перед боевым использованием это стоит поднять отдельно.

## Логика этапов

Список этапов и то, нужен ли на них сорт (QC), — жёстко задан в `src/db.js` (массив `STAGES`), чтобы никто случайно не сломал структуру отчёта. Если нужно добавить/переименовать этап — это правка кода (1 файл, 1 строка), а не через админку, в отличие от номенклатуры.
